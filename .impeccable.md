# Mergen — Impeccable Design Guide

## Design Context

### Users
Solo personal use by one developer/builder. Checked on mobile (Samsung Galaxy S25 Android) throughout the day — quick captures, status checks, adding todos, watching the movie bot. Desktop for deeper work. The user is the only user; there's no audience to impress, only a tool to trust.

### Brand Personality
**Calm, sharp, personal.**
Mergen is a private cockpit — not a product, not a SaaS dashboard. It knows its owner. It's quiet when there's nothing to say and precise when there is. The purple accent (`hsl(263 90% 65%)`) is the only personality it shows.

### Aesthetic Direction
- **Reference**: Linear — information-dense, fast, keyboard-first, elegant without trying. Lots of data per screen, nothing wasted.
- **Anti-reference**: Flashy / gamified — no badges, gradients-for-show, celebration animations, or startup-demo energy. No Notion-style white-block corporate feel.
- **Theme**: Dark-only. Background `hsl(0 0% 4%)`, cards `hsl(0 0% 9%)`, text `hsl(0 0% 95%)`.
- **Typography**: System UI stack (`-apple-system`, `BlinkMacSystemFont`, `Segoe UI`, `system-ui`). No decorative fonts.
- **Color**: Purple primary only (`hsl(263 90% 65%)`). Semantic status colors allowed when they answer "how worried/much should I be?" — red for overdue/destructive, amber/orange for medium effort or warning, green for completion or low effort. Category/type labels (Film, Lecture, Job, Anime) use muted unified style — the text carries the meaning. No rainbow palettes for decoration.
- **Motion**: Minimal. Functional transitions only (fade-in for dropdowns, subtle slide for panels). Never decorative.
- **Mobile**: Touch targets ≥ 44px. No hover-only interactions. Compact vertical rhythm — scrolling is friction.

### Design Principles

1. **Information density over empty space.** Show more per screen rather than padding things out. Especially on mobile, every pixel of vertical space is precious. Tight `gap-2`/`gap-3` on mobile, relaxed on desktop.

2. **Actions are always reachable.** Never hide *action controls* (edit, delete, move) behind hover — they must always be tappable. On mobile, use a ⋮ (MoreVertical) contextual menu. On desktop, hover-reveal for actions is acceptable. Touch targets must be large enough to tap confidently. **Exception**: hover-to-reveal *informational details* (e.g. movie card overlays with synopsis/ratings) is a desktop enhancement — additive context, not a required action. This is fine and encouraged on desktop.

3. **Calm consistency.** Stick to the established palette: purple primary, muted secondary text, card surfaces. Resist adding new colors for novelty. If something needs visual weight, use opacity or size — not color.

4. **Linear-style precision.** Borders, spacing, and type sizes should feel considered and tight — not loose. Prefer `rounded-lg` over `rounded-2xl`. Prefer subtle borders over elevated shadows.

5. **No noise.** Remove anything that doesn't serve a function. No decorative icons, no placeholder animations, no explainer copy the user already knows. The interface should disappear into the workflow.

## Tech Stack
- **Framework**: Next.js 16 (App Router) + Convex (backend/realtime)
- **Auth**: Clerk (dark theme configured)
- **Styling**: Tailwind CSS v4 (no config file — uses CSS variables in `globals.css`)
- **Components**: Custom, lucide-react icons, Radix UI primitives, dnd-kit (drag-and-drop), sonner (toasts), cmdk (command palette)
- **Font**: System UI stack
- **Design tokens** (CSS variables in `src/app/globals.css`):
  - `--background: 0 0% 4%`
  - `--foreground: 0 0% 95%`
  - `--card: 0 0% 9%`
  - `--border: 0 0% 22%`
  - `--primary: 263 90% 65%` (purple)
  - `--muted: 0 0% 15%`
  - `--muted-foreground: 0 0% 68%`
  - `--destructive: 0 70% 50%`
  - `--radius: 0.5rem`
